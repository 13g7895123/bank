## 解析器改善建議

### 問題分析

目前 `report_generator.py` 無法正確處理以下情況：

1. **表格中無現金卡項目**：解析器期望找到「現金卡」行，但部分銀行的表格直接省略此項
2. **數值為 `-` 的處理**：部分解析邏輯可能無法將 `-` 正確轉換為 0
3. **表格結構差異**：不同銀行的欄位數量和順序不同

### 建議修改

```python
# 1. 對於無現金卡項目的銀行，自動填入 0
def extract_from_table(table, year, quarter, bank_name):
    # ... existing code ...
    
    # 檢查是否有現金卡項目
    has_cash_card = any("現金卡" in str(row) for row in table)
    
    if not has_cash_card:
        # 自動補上現金卡項目，數值為 0
        results.append(AssetQualityRow(
            year=year,
            quarter=quarter,
            subject="04_消費金融_現金卡",
            overdue_amount=0,
            total_loan=0,
            overdue_ratio=0,
            bank_name=bank_name,
        ))

# 2. 對於數值為 '-' 的處理
def parse_number(value: str) -> float:
    if value is None:
```

## 結論

### 現金卡業務現況

根據調查結果，目前台灣銀行業的現金卡業務已大幅萎縮：

- **已完全無現金卡業務**：華南、渣打、台中、新光、永豐、凱基、安泰等銀行
- **有業務但餘額極小**：土銀（1,386仟元）、一銀（93仟元）、元大（7,873仟元）
- **有業務且餘額較大**：僅星展銀行（75,217仟元）

### 解析器處理建議

| 原始資料狀況 | 建議處理方式 | 輸出值 |
|--------------|--------------|--------|
| 表格無現金卡行 | 自動補 0 | `overdue=0, total=0, ratio=0` |
| 數值為 `-` | 轉換為 0 | `0` |
| 數值為 `0` | 直接使用 | `0` |
| 有實際數值 | 直接使用 | 實際數值 |

### 影響評估

即使解析器無法取得現金卡資料，對整體報表的影響極小：
- 現金卡餘額佔各銀行放款總額的比例通常 < 0.01%
- 多數銀行的現金卡業務已名存實亡或完全停止
